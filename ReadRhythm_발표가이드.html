슬라이드 1~2 · 도입 — 문제 정의 & 솔루션

1
왜 ReadRhythm인가? — 문제 정의
⏱ 약 1분
"독서는 하고 싶은데, 꾸준히 지속하는 게 어렵습니다. 오늘 30분 읽었는지, 이번 달 몇 권 읽었는지 — 막연히 느낌으로만 기억하죠. 도서관에서 자료를 찾고 싶을 때도 어디에 물어봐야 할지 막막합니다."
  • 독서 시간·권수를 기록하는 체계적인 앱의 부재
  • AI 추천은 있지만 나의 독서 패턴에 맞춘 추천은 드묾
  • 국립중앙도서관 Q&A — 전문적이지만 앱 안에서 바로 접근 불가
2
솔루션 — ReadRhythm의 3가지 핵심 가치
⏱ 약 1분
"ReadRhythm은 세 가지로 이 문제를 해결합니다. 기록하고, 질문하고, 성장한다 — 이 세 단어로 앱 전체를 설명할 수 있습니다."
  • 🗂️ 기록 — 타이머로 독서 시간 측정, 서재에 책 상태 관리, XP/레벨 시스템으로 동기 부여
  • 🤖 질문 — 국립중앙도서관 Q&A 5,907건 기반 AI 사서에게 언제든 질문, 현재 읽는 책 맥락 자동 인식
  • 📈 성장 — Claude AI가 독서 패턴을 분석해 맞춤 추천, 주간 코치 리포트로 성장 확인

💡 발표 팁

슬라이드 1-2는 청중이 "아, 나도 그런 경험 있어" 하고 공감할 수 있게 유도하는 것이 핵심입니다. 수치보다는 공감 스토리로 시작하세요.

Q&A 대비 — 예상 질문 & 모범 답변

RAG와 일반 ChatGPT의 차이가 뭔가요?
일반 GPT는 학습 데이터로만 답변하지만, RAG(Retrieval-Augmented Generation)는 실제 데이터베이스에서 관련 정보를 먼저 검색하고 그것을 근거로 답변을 생성합니다. 국립중앙도서관의 실제 사서 Q&A가 근거가 되기 때문에, 도서관 이용·자료 추천·저작권 등에서 훨씬 정확하고 신뢰할 수 있는 답변이 나옵니다.
5,907건 데이터는 어떻게 수집했나요?
국립중앙도서관 홈페이지의 디지털 사서 Q&A 게시판을 Python으로 크롤링했습니다. 질문-답변 쌍을 추출하고, sentence-transformers로 384차원 벡터로 변환해 PostgreSQL+pgvector에 저장했습니다. 임베딩 모델은 로컬에서 실행해 API 비용이 들지 않습니다.
Gemini와 Claude를 둘 다 쓴 이유가 뭔가요?
역할에 맞는 모델을 선택했습니다. Gemini 2.5 Flash는 스트리밍 속도가 빠르고 무료 티어가 있어 RAG 실시간 답변에 적합합니다. Claude Opus 4.7은 Extended Thinking으로 복잡한 추론이 필요한 독서 패턴 분석·추천에 사용했습니다. 각 AI의 강점을 역할에 맞게 배분한 것입니다.
실제 배포는 언제 하나요?
코드베이스는 EAS Build 설정까지 완료된 상태입니다. Apple Developer Account(연 129,000원)와 Google Play Console(25달러 1회) 가입 후 TestFlight·Play Console에 바로 배포 가능합니다. API 키(Gemini, Anthropic)와 PostgreSQL 클라우드 이전이 필요합니다.
혼자 개발했나요? 얼마나 걸렸나요?
네, 기획·설계·크롤링·DB 구축·프론트엔드·백엔드·AI 통합·GitHub 배포까지 전 과정을 혼자 진행했습니다. Claude Code AI 도구를 적극 활용해 12주 만에 완성했습니다. 화면 수 30개 이상, GitHub 레포 3개, Q&A 데이터 5,907건 구축이 포함됩니다.
