🟢 도입부 (약 1~2분)

🎤 발표 대본 0:00 ~ 1:30
안녕하십니까. 오늘은 제가 개발한 ReadRhythm, 'AI 기반 독서 습관 트래커'를 소개드리겠습니다. 여러분, 이런 경험 있으신가요? 독서는 하고 싶은데, 막상 꾸준히 지속하는 게 어렵습니다. 오늘 30분 읽었는지, 이번 달 몇 권 읽었는지 — 막연히 느낌으로만 기억하고, 기록이 남지 않습니다. 또, 좋은 책을 추천받고 싶을 때 어디에 물어봐야 할지 막막합니다. 인터넷 검색은 너무 광범위하고, 도서관 사서 선생님께 매번 물어볼 수도 없습니다. ReadRhythm은 이 두 가지 문제를 한 앱으로 해결합니다. 첫째, 독서를 기록하고 관리합니다. 타이머로 시간을 재고, 서재에 책을 정리하고, 레벨과 배지로 성취감을 줍니다. 둘째, 국립중앙도서관의 실제 사서 질문·답변 5,910건을 AI로 학습시켜, 앱 안에서 언제든 전문적인 도서 상담을 받을 수 있게 했습니다.
💡 "5,910건"이라는 숫자를 자신 있게 강조하세요. 단순한 AI가 아닌 실제 사서의 지식이라는 점이 핵심 차별점입니다.

🟢 기술 스택 설명 (약 2분)

🎤 발표 대본 1:30 ~ 3:30
앱 기술에 대해 간단히 설명드리겠습니다. 앱은 React Native Expo라는 기술로 만들었습니다. 쉽게 말씀드리면, 코드 하나로 아이폰, 안드로이드, 웹 브라우저에서 모두 동작하는 방식입니다. 일반적으로 아이폰 앱과 안드로이드 앱을 따로 만들어야 하는데, 이 기술 덕분에 하나만 만들면 세 곳에서 모두 돌아갑니다. 뒤에서 서버 역할을 하는 FastAPI는 Python 언어로 만든 웹 서버입니다. 식당으로 비유하면, 앱은 손님이 주문하는 홀이고, FastAPI는 실제로 요리를 만드는 주방입니다. 여기에 AI를 두 가지 붙였습니다. Gemini는 구글이 만든 AI로, 사서 질문에 빠르게 스트리밍으로 답변을 생성합니다. Claude Opus 4.7은 Anthropic이 만든 AI로, 내 독서 패턴을 분석해 맞춤 책을 추천하고 주간 코치 리포트를 작성합니다. 데이터는 PostgreSQL이라는 데이터베이스에 저장됩니다. 이 데이터베이스에 pgvector라는 확장 기능을 더해서, 사서 Q&A 5,910건을 의미 단위로 검색할 수 있게 만들었습니다. — 이 부분은 뒤에서 더 자세히 설명드리겠습니다.
💡 기술 이름을 나열할 때 무조건 "간단히 말씀드리면..."으로 비유를 붙여 주세요. 청중이 이름을 못 외워도 괜찮습니다.

🟢 v1.0 앱 데모 — 5탭 투어 (약 3분)

🎤 발표 대본 3:30 ~ 6:30 ◀ [앱 화면 열기]
직접 앱을 보여드리겠습니다. 먼저 홈 화면에서 '데모 채우기' 버튼을 눌러 실제 사용 중인 상태로 만들겠습니다. (홈 화면 — 데모 채우기 클릭) "오늘 목표 35분 중 35분 달성, 채식주의자를 읽는 중, 6일 연속 독서 중" 이렇게 오늘의 현황이 자동으로 표시됩니다. 꾸준히 읽으면 이렇게 기록이 쌓입니다. (서재 탭 클릭) 서재에서는 책을 세 가지 상태로 분류합니다. 읽는 중, 완독, 읽고 싶어요. 오른쪽에 장르 도넛 차트가 있어서 내가 어떤 장르를 주로 읽는지 한눈에 볼 수 있습니다. (타이머 탭 클릭) 타이머는 3중 링 모양으로 되어 있습니다. 바깥 링은 목표 시간, 가운데 링은 실제 읽은 시간, 안쪽 링은 페이지 진행입니다. 자유 모드와 25분 집중 모드를 선택할 수 있습니다. (내정보 탭 클릭) 내정보 탭에서는 레벨, 배지, 연간 리포트를 볼 수 있습니다. 지금은 Lv.2 책 애호가, 배지 4개 획득, 6일 스트릭이 쌓여 있습니다. 30종의 배지가 있고, 독서왕이 최고 레벨입니다.
💡 탭을 클릭할 때 1초 멈춰서 화면이 뜨는 걸 보여준 뒤 설명하세요. 너무 빨리 넘기면 청중이 따라오기 어렵습니다.

🟢 AI 사서 시연 (약 2분)

🎤 발표 대본 6:30 ~ 8:30 ◀ [AI사서 탭 클릭]
이제 이 앱의 핵심인 AI 사서 기능을 보여드리겠습니다. 탭을 열면 상단에 "지금 읽는 책: 채식주의자 (한강)" 라는 바가 보입니다. 서재에서 읽고 있다고 등록한 책을 AI가 자동으로 알아채서 표시해줍니다. 그리고 아래에 세 개의 버튼이 자동 생성됩니다. "한강 저자의 다른 작품 추천", "채식주의자 관련 자료 찾기", "채식주의자와 비슷한 책" 이 버튼 중 하나를 누르거나, 직접 질문을 입력하면 됩니다. (질문 전송) AI가 5,910건의 사서 Q&A에서 가장 관련 있는 내용을 찾아서 Gemini가 자연스러운 말로 정리해서 답해줍니다. 중요한 점은, AI가 상상으로 지어낸 게 아니라 실제 사서가 답변한 내용이 근거가 된다는 것입니다. 그래서 도서관 이용, 자료 추천, 저작권 문의 등에서 특히 신뢰할 수 있는 답변이 나옵니다. 답변 아래로 스크롤하면 "언급된 도서" 섹션이 자동으로 나타납니다. AI 답변 속에서 책 제목을 자동으로 뽑아내서 보여주고, 각 책 옆에 "+ 담기" 버튼이 있습니다. 누르면 서재에 바로 추가됩니다.
💡 로딩 중에는 "지금 5,910건에서 관련 답변을 찾고 있습니다"라고 설명하면 자연스럽습니다. 느려도 당황하지 마세요.

🟢 AI 추천 & 코치 리포트 (약 1분)

🎤 발표 대본 8:30 ~ 9:30
마지막으로 AI 도서 추천과 코치 리포트입니다. 홈 화면에서 'AI 맞춤 추천' 버튼을 누르면, Claude AI가 내 서재와 독서 세션 기록을 분석해서 책 5권을 추천합니다. 단순 추천이 아니라 책마다 나와의 궁합 퍼센트와 추천 이유 3가지, 예상 독서 완료일까지 계산해서 보여줍니다. 코치 리포트는 내정보 탭에서 볼 수 있습니다. 이번 주 독서 시간, 전주 대비 성장률, 강점 3가지와 개선 팁을 AI가 분석해줍니다. 가장 열심히 읽었던 시간대, 선호 장르 같은 패턴도 파악해서 알려줍니다.
💡 Claude AI는 실제 데모 중에 느릴 수 있습니다. 미리 결과를 캡처해두거나, 목업 데이터로 보여줘도 자연스럽습니다.

✨ v2.0 신규 기능 (약 3~4분)

🎤 발표 대본 9:30 ~ 13:00 v2.0 NEW
v1.0 12주 MVP 완성 후, 실제 사용성을 높이기 위해 네 가지 개선을 더 진행했습니다. 첫 번째, AI 사서 답변에서 책을 바로 서재에 담기 (Task #4) 아까 보셨던 "+ 담기" 버튼입니다. 답변 텍스트에서 책 제목을 자동으로 추출해서, 클릭 한 번으로 서재에 연결됩니다. AI 대화 → 서재 연동을 원클릭으로 만든 것입니다. 혹시 AI 추출이 안 될 때를 대비해 정규식이라는 문자 패턴 인식 방법으로 최대 6권까지 보조 추출합니다. 두 번째, 백엔드를 하나로 통합 (Task #5) v1.0에서는 AI 추천을 담당하는 서버와 AI 사서 서버가 따로 돌아갔는데, v2.0에서는 FastAPI 하나로 모두 통합했습니다. 관리가 훨씬 단순해지고 안정성이 올라갔습니다. 세 번째, 현재 읽는 책 컨텍스트 바 (Task #6) 아까 AI사서 탭 상단에 보였던 "지금 읽는 책: 채식주의자" 바입니다. 서재에서 읽는 중인 책을 자동으로 인식해서 맞춤 질문 버튼을 만들어줍니다. 사용자가 별도로 설정할 필요 없이 자동으로 동작합니다. 네 번째, 5탭 네비게이션 재편 (Task #7) 원래 탭이 7개였는데, 핵심 5개만 남기고 BGM과 문장보관함은 타이머와 내정보 탭의 단축 버튼으로 접근할 수 있게 바꿨습니다. 화면이 더 깔끔해지고 집중도가 높아졌습니다.
💡 v2.0 기능은 이미 AI사서 데모에서 자연스럽게 보여줬으므로, 이 대본에서는 "아까 보셨던"이라고 연결하면 흐름이 좋습니다.

🟢 마무리 (약 1분)

🎤 발표 대본 13:00 ~ 14:00
정리하겠습니다. ReadRhythm은 두 가지를 합쳤습니다. 하나는 독서 습관 관리 — 타이머, 서재, 레벨, 배지로 독서를 꾸준히 이어가게 합니다. 다른 하나는 AI 사서 — 국립중앙도서관의 실제 사서 답변 5,910건을 학습시켜서 전문 사서에게 물어보는 경험을 앱 안에서 언제든 제공합니다. v1.0 12주 동안 기초를 완성했고, v2.0에서 답변 내 책 담기, 컨텍스트 바, 백엔드 통합, 탭 재편으로 사용성을 높였습니다. 기록하고, 질문하고, 성장한다. 이 세 단어가 ReadRhythm입니다. 감사합니다.
💡 마지막 세 단어 "기록하고, 질문하고, 성장한다"는 천천히 한 단어씩 끊어서 말하면 인상적입니다.

데모 ① — 홈 화면 + 서재 + 타이머 (v1.0 핵심)

  1. 브라우저에서 http://localhost:8081 열기 → 홈 화면 표시됨
    → 목표 진행바 · 오늘의 독서 현황 · 연속 독서 스트릭 표시

    → "채식주의자 읽는 중, 6일 스트릭, 오늘 35분 완료" 상태로 채워짐
  3. 통계 수치 가리키며: "오늘 목표 달성, 6일 연속 독서 중입니다."
  4. 하단 탭에서 📚 서재 탭 클릭
    → 책 목록 + 오른쪽 장르 도넛 차트
  5. 서재 책 하나 클릭 → 상세 화면 보여주기 → 뒤로가기
  6. 하단 탭에서 ⏱ 타이머 탭 클릭
    → 3중 링 SVG 타이머 표시
  7. 링 가리키며: "바깥 링은 목표, 가운데는 실제 시간, 안쪽은 페이지입니다."

데모 ② — AI 사서 + 컨텍스트 바 + 서재 담기 (v2.0 하이라이트)

  1. 하단 탭에서 🤖 AI사서 탭 클릭
    → 상단에 "지금 읽는 책: 채식주의자 (한강)" 컨텍스트 바 자동 표시
  2. 컨텍스트 바 가리키며: "서재에 등록한 책을 자동으로 인식해서 보여줍니다."
  3. 제안 칩 중 "중학생을 위한 과학책 추천" 클릭 (또는 직접 입력)
    → AI가 스트리밍으로 답변 생성 (글자가 한 글자씩 나타남)
  4. 로딩 중 멘트: "지금 5,910건의 사서 Q&A에서 관련 내용을 찾아 정리하고 있습니다."
  5. 답변 완료 후 아래로 스크롤
    → "언급된 도서" 섹션과 책 목록이 자동 표시됨
  6. 책 옆 + 담기 버튼 클릭
    → 토스트 메시지: "서재에 추가되었습니다"
  7. 서재 탭으로 이동해서 추가된 책 확인 (선택)

데모 ③ — 내정보 탭 (배지 · 레벨 · 단축 메뉴)

  1. 하단 탭에서 👤 내정보 탭 클릭
    → 레벨 · 배지 · 이번 달 달성일 캘린더
  2. 레벨 카드 가리키며: "Lv.2 책 애호가, 배지 4개 획득, 연속 6일 독서입니다."
  3. 배지 그리드 가리키며: "30종의 배지가 있고, 조건을 달성하면 자동으로 해금됩니다."
  4. 메뉴 그리드에서 🎵 BGM 단축 버튼 가리키며: "BGM도 이 탭에서 바로 진입할 수 있습니다."

데모 ④ — AI 도서 추천 (선택 · 시간 여유 있을 때)

  1. 홈 화면 → AI 맞춤 추천 버튼 클릭
    → Claude AI 분석 중 (약 5~15초 소요)
  2. 추천 카드 표시 후 가리키며: "궁합 퍼센트, 추천 이유 3가지, 예상 완료일이 책마다 나옵니다."
  3. 내정보 → 코치 리포트 메뉴 클릭 → 주간 분석 결과 보여주기
추천 데모 순서: ① 홈+서재+타이머 → ② AI사서+담기 (핵심) → ③ 내정보
시간이 부족하면 ②번만 해도 충분합니다. AI 사서 + "+ 담기" 데모가 가장 인상적입니다.
3 돌발 상황 대처법 당황하지 말고 이 대응 스크립트를 쓰세요
🚨 상황 1 — AI 사서가 느리거나 응답이 없을 때
Gemini AI 무료 사용 한도(하루 20회)를 초과했거나 네트워크가 느릴 때
어떻게 되나요?: 할당량 초과 시 DB 검색 결과를 그대로 표시하는 백업 모드로 자동 전환됩니다.
도서 추출도 정규식(문자 패턴 인식)이 자동으로 대신합니다. 앱이 멈추지 않습니다.
사전 예방: 발표 당일 아침에 AI 사서 탭에서 짧은 질문 하나로 상태 확인.
🎤 대본: "지금 AI가 응답을 처리하고 있습니다. 5,910건에서 찾는 중이니 잠깐만 기다려주세요."
→ 30초 후에도 안 오면: "AI 응답 속도가 조금 느리네요. 다른 기능을 먼저 보여드리겠습니다." → 내정보 탭으로 전환
🚨 상황 2 — "+ 담기" 버튼 눌렀는데 도서 목록이 안 나올 때
Gemini 할당량 초과 + 정규식도 책을 추출하지 못한 경우
원인: 답변에 명확한 책 제목이 없거나, 외국어 제목 등 패턴이 다른 경우.
대처: 자연스럽게 넘어가세요. "언급된 도서" 섹션이 없어도 사서 답변 자체는 정상입니다.
🎤 대본: "이 질문의 답변에는 특정 책보다는 방법을 안내하는 내용이 많아서 자동 추출이 생략됐습니다. 책 제목이 포함된 추천 질문에서는 바로 '+ 담기' 버튼이 나타납니다."
→ 미리 준비한 "중학생 과학책 추천" 질문으로 다시 시연
🚨 상황 3 — FastAPI 서버가 응답하지 않을 때 (서버 오류)
AI사서 탭에서 "서버에 연결할 수 없습니다" 메시지가 나올 때
원인: FastAPI 서버가 실행 중이지 않거나 재시작이 필요한 상태.
대처: 터미널에서 서버 재실행. AI 추천은 목업 데이터로 자동 폴백되어 정상 표시됩니다.
🎤 대본: "서버 연결을 잠깐 확인하겠습니다. 그동안 앱의 다른 기능을 먼저 보여드리겠습니다."
→ 타이머 또는 서재 탭으로 전환 → 터미널에서 uvicorn 재실행
🚨 상황 4 — "그냥 독서 앱 쓰면 되지 않나요?" 질문을 받을 때
좋은 질문입니다. 기존 앱과의 차별점을 구체적으로 답하세요.
🎤 대본: "맞습니다, 독서 기록 앱은 시중에 여러 개 있습니다. ReadRhythm은 거기에 두 가지를 더합니다. 첫 번째는 AI 사서입니다. 기존 앱에는 없는 기능으로, 국립중앙도서관 5,910건의 실제 사서 답변을 학습시켜서 도서관 수준의 전문 상담을 앱 안에서 받을 수 있습니다. 두 번째는 읽는 책 맥락 인식입니다. 내가 지금 읽는 책을 AI가 파악해서 관련 질문 버튼을 자동으로 만들어줍니다. 이 조합은 다른 앱에 없는 기능입니다."
🚨 상황 5 — "AI가 틀린 정보를 줄 수도 있지 않나요?" 질문
AI 신뢰성에 대한 합리적 질문입니다. RAG 구조를 쉽게 설명하세요.
🎤 대본: "맞습니다. 일반 AI는 학습 데이터를 바탕으로 답을 만들어내기 때문에 틀릴 수 있습니다. ReadRhythm의 AI 사서는 RAG라는 방식을 사용합니다. 쉽게 말씀드리면, AI가 답을 지어내는 게 아니라 국립중앙도서관 실제 사서가 답변한 5,910건의 데이터에서 관련 내용을 먼저 찾아내고, 그걸 바탕으로 정리해서 답합니다. 출처가 있는 답변이기 때문에 일반 AI보다 훨씬 신뢰도가 높습니다."
🚨 상황 6 — 핵심 수치가 갑자기 기억나지 않을 때
"5,910건이었나 5,190건이었나..." 하고 헷갈리는 순간이 올 수 있습니다.
🎤 대본: "정확한 수치를 확인하겠습니다." → AI사서 탭 상단 헤더 텍스트 확인
(AI사서 화면 헤더에 "실제 사서 답변 5,910건을 근거로 응답합니다"라고 표시됩니다. 자연스럽게 화면을 "확인"하는 것처럼 보이면 됩니다.)

🔵 AI · 검색 관련 용어

RAG (검색 증강 생성)
Retrieval-Augmented Generation
AI가 답을 지어내는 대신, 먼저 데이터베이스에서 관련 자료를 검색한 뒤 그 자료를 근거로 답변을 만드는 방식. 근거 있는 답변이 나온다.
📚 오픈북 시험 — 교재를 찾아보고 답변하는 것 (암기X)
임베딩 (Embedding)
단어·문장의 의미를 숫자 배열로 변환
컴퓨터가 단어의 "의미"를 이해하도록 숫자로 변환하는 과정. "독서"와 "책 읽기"는 비슷한 숫자, "요리"는 전혀 다른 숫자로 표현된다.
📚 단어에 GPS 좌표를 부여 — 의미가 비슷하면 좌표도 가까움
벡터 검색
Vector Search — 의미 유사도 기반 검색
단어가 정확히 일치하지 않아도 의미가 비슷하면 찾아주는 검색. "좋은 책 알려줘"라고 해도 "도서 추천"이라는 답변을 찾아낸다.
📚 지도에서 가장 가까운 식당 찾기 — 이름이 아닌 거리로 검색
BM25 (키워드 검색)
Best Match 25 — 단어 빈도 기반 검색
정확한 단어가 얼마나 자주, 얼마나 드물게 나오는지로 점수를 계산해 관련 문서를 찾는 방식. 단어가 정확히 맞을 때 강함.
📚 교과서 색인(찾아보기) — 단어를 그대로 입력하면 페이지를 찾아줌
하이브리드 검색
키워드 검색 + 의미 검색 동시 실행
BM25와 벡터 검색을 동시에 하고 결과를 합산. 키워드가 정확할 때도, 의미가 비슷할 때도 모두 잘 찾아낸다. ReadRhythm은 벡터 65% + BM25 35%로 혼합 사용.
📚 필기 + 면접을 종합해 최종 합격자 선발
sentence-transformers
문장 임베딩 생성 오픈소스 모델
문장 전체의 의미를 숫자 배열(벡터)로 변환해 주는 프로그램. 무료로 컴퓨터 안에서 실행해 API 비용이 들지 않는다. ReadRhythm에서 5,910건 전체를 변환하는 데 사용.
📚 무료 사내 번역가 — 외주 없이 직접 처리
Gemini 2.5 Flash
Google DeepMind의 AI 언어 모델
Google이 만든 AI. 빠른 응답 속도와 스트리밍(글자가 실시간으로 나타남)이 강점. ReadRhythm에서 사서 Q&A 답변 생성과 도서 추출에 사용.
📚 속기사 — 빠르게 타이핑하며 실시간으로 답변 전달
Claude Opus 4.7
Anthropic의 AI 언어 모델
Anthropic이 만든 고급 AI. 복잡한 추론에 강점. ReadRhythm에서 독서 패턴 분석·맞춤 책 추천·코치 리포트 작성에 사용. Extended Thinking 기능으로 더 깊이 생각한다.
📚 심층 분석 컨설턴트 — 시간을 들여 깊이 있게 분석
Extended Thinking
AI가 답하기 전에 내부적으로 더 깊이 추론하는 기능
Claude AI가 즉시 답하는 대신, 내부적으로 단계별 추론을 더 거친 후 답변을 내놓는 방식. 복잡한 분석에서 더 정확하고 논리적인 결과가 나온다.
📚 초안을 잡고 여러 번 다듬어서 최종 보고서 작성
SSE (스트리밍)
Server-Sent Events — 서버에서 클라이언트로 실시간 전송
AI 답변이 다 완성될 때까지 기다리는 대신, 한 글자씩 실시간으로 보내주는 방식. ChatGPT가 글자를 타이핑하듯 보여주는 것이 이 방식.
📚 편지 전체 완성 후 보내기 vs. 전화로 실시간 말하기

🟣 데이터베이스 관련 용어

PostgreSQL (포스트그레스)
오픈소스 관계형 데이터베이스
데이터를 표(테이블) 형태로 저장하고 관리하는 무료 데이터베이스 소프트웨어. ReadRhythm에서 사서 Q&A 5,910건과 임베딩 벡터를 저장.
📚 데이터를 담는 거대한 엑셀 파일 — 동시 접속·검색에 최적화
pgvector
PostgreSQL용 벡터 검색 확장 플러그인
PostgreSQL에 벡터(숫자 배열) 저장·검색 기능을 추가해 주는 플러그인. 이것 덕분에 의미 기반 검색을 데이터베이스 안에서 직접 처리할 수 있다.
📚 일반 창고에 냄새 맡는 탐지견 추가 — 내용까지 검색 가능
HNSW 인덱스
Hierarchical Navigable Small World — 계층적 벡터 탐색 자료구조
5,910건 중 가장 유사한 벡터를 빠르게 찾기 위한 자료구조. 전체를 다 비교하는 대신 계층적으로 좁혀가며 검색해 속도가 훨씬 빠름.
📚 도서관 분류 체계 — 전체를 다 뒤지지 않고 구역→책장→책 순으로 좁혀감
GIN 인덱스
Generalized Inverted Index — 전문 검색 인덱스
형태소(단어의 어근)를 기반으로 빠른 키워드 검색을 가능하게 하는 색인. BM25 검색 속도 향상에 사용.
📚 백과사전 뒤의 색인 목록 — 단어 → 페이지 번호 즉시 찾기
kiwipiepy (키위파이파이)
한국어 형태소 분석기 라이브러리
한국어 문장을 단어의 어근 단위(형태소)로 분리해 주는 파이썬 라이브러리. "읽었다"를 "읽-+-었-+-다"로 분리해서 키워드 검색 정확도를 높인다.
📚 국어사전 편집자 — 단어를 가장 작은 의미 단위로 나눔
크롤링 (Crawling)
웹 데이터를 자동으로 수집하는 프로그램
국립중앙도서관 Q&A 게시판에서 질문·답변 데이터를 Python 프로그램이 자동으로 5,910건을 수집한 방법. 사람이 하나씩 복사·붙여넣기하는 대신 자동화.
📚 도서관 전 권을 자동으로 사진 찍어 디지털화하는 기계

🟢 앱 · 서버 관련 용어

React Native
Facebook이 만든 크로스플랫폼 앱 개발 프레임워크
하나의 코드로 iOS(아이폰), Android(갤럭시), 웹 브라우저에서 동시에 실행되는 앱을 만들 수 있는 기술. 보통 세 곳에 별도로 만들어야 하지만 이것으로 한 번에 가능.
📚 다국어 동시 번역기 — 하나로 여러 나라에서 읽힘
Expo / Expo SDK 54
React Native 개발 편의 도구 모음
React Native 앱을 더 쉽게 만들 수 있게 도와주는 도구. 카메라·저장소·알림 같은 기능을 간단하게 쓸 수 있게 제공한다. SDK 54는 54번째 버전.
📚 조립식 가구 부품 세트 — 기본 재료를 미리 다 갖춰줌
TypeScript
JavaScript에 타입 안전성을 추가한 언어
JavaScript라는 프로그래밍 언어에 "이 변수는 반드시 숫자여야 한다"는 규칙을 추가한 버전. 실수를 미리 잡아주어 코드 오류가 줄어든다.
📚 맞춤법 검사기가 내장된 워드 프로그램
Zustand (주스탄트)
경량 전역 상태관리 라이브러리
앱 전체에서 공유해야 하는 데이터(서재 목록, 레벨, 설정 등)를 한 곳에서 관리하는 도구. 코드가 간결하고 빠르다.
📚 앱 전체가 공유하는 공동 메모장
AsyncStorage
React Native의 앱 내부 로컬 저장소
앱을 종료해도 데이터가 지워지지 않도록 기기에 저장하는 공간. 서재 목록, 레벨, 설정 등이 여기 저장되어 앱을 껐다 켜도 유지된다.
📚 냉장고 — 앱이 꺼져도 내용물이 그대로 보존됨
TanStack React Query
서버 데이터 캐싱·동기화 라이브러리
서버에서 받아온 데이터를 앱 안에 임시 저장(캐시)해서 매번 새로 불러오지 않아도 되게 하는 도구. 빠르고 데이터가 자동으로 최신 상태를 유지한다.
📚 자주 쓰는 서류는 책상 위에 꺼내두기 — 매번 캐비닛 열지 않아도 됨
FastAPI
Python 기반 고성능 웹 API 서버 프레임워크
Python 언어로 서버를 빠르게 만들 수 있는 도구. 앱의 요청을 받아서 AI·데이터베이스를 연결하고 결과를 돌려주는 역할. v2.0에서 단일 서버로 통합.
📚 주방 — 손님(앱) 주문을 받아 요리(AI 처리)해 서빙
Expo Router
파일 구조 기반 자동 화면 연결 시스템
파일 이름만으로 앱 화면 경로가 자동 설정되는 방식. 예: "app/shelf.tsx" 파일을 만들면 자동으로 서재 화면이 된다. 별도 경로 설정 코드 불필요.
📚 폴더 이름이 그대로 목차가 되는 보고서 자동화
Reanimated + SVG
React Native 고성능 애니메이션 + 벡터 그래픽
Reanimated: 타이머 링이 부드럽게 돌아가게 해주는 애니메이션 도구. SVG: 타이머 3중 링, 장르 차트 등 확대해도 선명한 그래픽을 그리는 방식.
📚 시침·분침이 부드럽게 돌아가는 시계
EAS Build
Expo Application Services — 앱 빌드·배포 클라우드 서비스
Expo 앱을 실제 앱스토어(Apple App Store, Google Play)에 올릴 수 있는 설치 파일(.ipa, .apk)로 변환해 주는 서비스. ReadRhythm은 설정 완료 상태.
📚 Word 파일 → 인쇄소 최종 출력 — 실제 배포 가능한 형태로 변환

🟡 기타 용어

API (에이피아이)
Application Programming Interface — 서비스 간 통신 규약
서로 다른 프로그램이 정해진 방식으로 데이터를 주고받는 약속. 예: 앱이 "책 추천 주세요"라고 보내면 서버가 "여기 5권입니다"라고 돌려주는 규칙.
📚 주문 양식 — 정해진 형식으로 주문하면 정해진 형식으로 답이 옴
MVP (엠브이피)
Minimum Viable Product — 최소 기능 제품
모든 기능을 다 넣은 완성품이 아니라, 핵심 기능만 먼저 만들어 테스트하는 초기 버전. ReadRhythm v1.0이 12주 MVP였고, v2.0에서 추가 개선.
📚 정식 출시 전에 시식 행사 — 핵심 맛만 먼저 확인
GitHub (깃허브)
코드 버전 관리·저장·공유 플랫폼
코드를 저장하고, 변경 이력을 관리하고, 온라인에서 공유할 수 있는 서비스. 잘못된 코드를 이전 버전으로 되돌리는 것도 가능. ReadRhythm은 레포 3개 관리 중.
📚 코드용 구글 드라이브 + 문서 변경이력 추적
ISBN (이에스비엔)
International Standard Book Number — 국제 표준 도서 번호
세계 모든 책에 부여된 고유 번호. 13자리 숫자로 구성. ReadRhythm에서 ISBN 숫자를 해시(특수 계산)해서 책마다 다른 색상의 커버를 자동 생성하는 데 사용.
📚 책의 주민등록번호 — 전 세계 어느 책도 중복 없이 구별
XP (경험치) / 레벨 시스템
Experience Points — 게임 요소를 앱에 도입한 성장 시스템
독서 세션을 완료하면 XP(경험치)를 받고, 일정량이 쌓이면 레벨이 올라간다. 총 7단계: 새싹→책벌레→독서가→지식인→사색가→탐구자→독서왕.
📚 독서 RPG 게임 — 책을 읽을수록 캐릭터가 성장
정규식 폴백 (Regular Expression Fallback)
패턴 매칭 기반 보조 추출 방식
AI가 책 제목 추출에 실패할 때 자동으로 대신 작동하는 보조 방법. 텍스트에서 『』, **, • 같은 패턴을 찾아서 책 제목을 추출. 최대 6권까지 인식.
📚 자동문이 고장났을 때 작동하는 수동 보조 손잡이
