1. 프로젝트 개요

📌 개발 배경 및 목표

독서는 지식 습득과 자기계발의 핵심 활동이지만, 꾸준한 독서 습관을 유지하기 어렵다는 문제가 있다. ReadRhythm은 독서 타이머·서재 관리·AI 사서 질의응답을 하나의 앱에 통합하여 독서 루틴의 형성과 지속을 돕는다.

핵심 차별점

국립중앙도서관의 실제 사서 Q&A 5,910건을 기반으로 한 RAG 시스템으로, 단순 AI 챗봇이 아닌 전문 사서의 지식을 앱 안에서 제공한다. v2.0부터는 현재 읽는 책을 인식하여 맞춤형 제안 칩을 자동 생성하고, 답변 내 책을 즉시 서재에 담을 수 있다.

3. 개선 & 확장 기능 — 2026-05-27 업데이트

Task #4 AI 답변 내 도서 자동 추출 & "+ 담기"
목적: AI사서 답변에서 언급된 책을 바로 서재에 담는 원클릭 연동
  • Gemini API로 답변 텍스트에서 도서 목록 JSON 구조화 추출
  • "언급된 도서" 섹션 자동 생성 — 책 제목 · 저자 표시
  • 각 도서마다 "+ 담기" 버튼 → 서재 '읽고 싶어요' 즉시 추가
  • 정규식 폴백 (Gemini 할당량 초과 시): 최대 6권 자동 추출
    • 『제목』 / **제목** (마크다운 볼드)
    • • 제목 : 부제 / 저자 지음 (국립중앙도서관 Q&A 형식)
Task #5 백엔드 단일화 (NestJS → FastAPI)
목적: 이원화된 서버 구조를 FastAPI 단일 백엔드로 통합하여 유지보수성 향상
  • NestJS(포트 3001) 기능 → FastAPI(포트 8000)로 전량 이전
  • 이전 엔드포인트: /api/ai/recommendations, /api/ai/coach-report
  • 환경변수 EXPO_PUBLIC_LIBRARIAN_URL 단일화 (이전 EXPO_PUBLIC_API_URL 폐기)
  • ReadRhythm-backend(NestJS) 레포 → archived 처리
BEFORE (v1.0)
FastAPI 포트 8000 (RAG)
NestJS 포트 3001 (AI추천)
환경변수 2개 관리
AFTER (v2.0)
FastAPI 포트 8000 (전체)
단일 백엔드
환경변수 1개 통합
Task #6 현재 읽는 책 컨텍스트 바
목적: 서재에서 읽는 책을 AI사서가 자동 인식하여 맥락 있는 질문 유도
  • shelfStore의 status === 'reading' 책 자동 감지
  • AI사서 탭 상단 컨텍스트 바 자동 표시 (책 제목 + 저자)
  • 맞춤 제안 칩 3개 자동 생성:
    • "[저자] 저자의 다른 작품 추천"
    • "[책 제목] 관련 자료 찾기"
    • "[책 제목]과 비슷한 책 추천"
  • 칩 클릭 → 해당 질문 자동 전송 (원탭 질의)
  • 읽는 책 없을 경우 컨텍스트 바 자동 숨김
Task #7 5탭 네비게이션 재편
목적: 핵심 탭을 5개로 집중하고 BGM·문장보관함은 단축 메뉴로 접근성 유지
  • 탭바에서 숨김: BGM 탭, 문장보관함 탭 (href: null — 라우트 유지)
  • 타이머 탭: 우측 상단 🎵 BGM 단축버튼 추가
  • 내정보 탭: 문장보관함 · BGM 단축 메뉴 그리드 추가
  • 최종 탭 구조: 🏠 홈 · 📚 서재 · ⏱ 타이머 · 🤖 AI사서 · 👤 내정보

내정보·BGM·문장보관함
